x-cabeceras, los X-Headers son una forma de agregar información adicional a las respuestas HTTP de un servidor web. Esta información puede ser utilizada para mejorar la seguridad del sitio web, proteger contra ataques y mejorar la experiencia del usuario.

Ubicación de los archivos de configuración X-Header de seguridad para Apache2:

Apache 2.4: /etc/apache2/apache2.conf
Apache 2.2: /etc/apache2/conf.d/security

X-Header más comunes:
X-XSS-Protection: Protege contra ataques de secuencias de comandos entre sitios (XSS).
X-Frame-Options: Protege contra ataques de clickjacking.
X-Content-Type-Options: Protege contra MIME sniffing.
Strict-Transport-Security (HSTS): Obliga a los navegadores a usar HTTPS para acceder al sitio web.
Content-Security-Policy (CSP): Define qué recursos pueden cargar los navegadores desde un sitio web.
Referrer-Policy: Controla cómo se envía la información del referrer a otros sitios web.

Apache
# Habilitar módulo headers, mod_headers: 
El módulo mod_headers es necesario para agregar X-Headers a las respuestas HTTP. Para habilitarlo, edite el archivo de configuración principal de Apache (/etc/apache2/apache2.conf para Apache 2.4 o /etc/apache2/conf.d/security para Apache 2.2) y busque la siguiente línea:
LoadModule headers_module modules/mod_headers.so

# Establecer X-XSS-Protection
Header set X-XSS-Protection "1; mode=block"

# Establecer X-Frame-Options
Header set X-Frame-Options "SAMEORIGIN"

# Establecer X-Content-Type-Options
Header set X-Content-Type-Options "nosniff"

# Establecer HSTS
Header always set Strict-Transport-Security "max-age=31536000; includeSubDomains; preload"

# Establecer CSP
Header set Content-Security-Policy "default-src 'self'; script-src 'self' https://ajax.googleapis.com; style-src 'self' https://fonts.googleapis.com; img-src 'self' data:; font-src 'self' https://fonts.gstatic.com"

# Establecer Referrer-Policy
Header set Referrer-Policy "no-referrer"

Configurar las directivas X-Header:
Header set X-XSS-Protection "1; mode=block"
